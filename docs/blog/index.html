Research Notes and Series

The CAISI blog is organized as collections, not one-off pages. The main 10-part operating-model series explains how governed AI engineering should work across repos and teams. Separate report series extract focused lessons from measured work: OpenClaw for governed runtime behavior, and sprawl for public AI adoption, approval visibility, and evidence posture.

How to read the blog

Each collection has a different job. The operating-model series builds the general framework. The OpenClaw series stays tightly tied to one runtime case study. The sprawl series translates a public measurement report into specific lessons for AppSec, CISO, and platform audiences. All three use the same site system: shared nav, breadcrumbs, and series landing pages.

Recurring themes

Systems over prompting

The unit of analysis is not prompt quality. It is the system that turns work into changes, approvals, evidence, and recovery paths.

Security and velocity can align

When workflows are designed correctly, the same controls that reduce blast radius also remove ambiguity and scale throughput.

Deterministic code owns deterministic steps

The best agent workflows interleave reasoning with scripts, validators, and policies that do not depend on model behavior.

Proof is part of the product

A governed system produces a reviewable packet from trigger to merge. Passing CI is necessary, but never sufficient.
